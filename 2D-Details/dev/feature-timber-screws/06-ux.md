# 06 — UX

> Interaction patterns. How the engineer goes from blank canvas to a checked, dimensioned detail. Pairs with `05-architecture.md` (the tools layer wires these up).

The interaction goal: an engineer should produce a fully‑checked HBS connection detail in under 60 seconds of clicking, with no menu hunting.

---

## 1. The canonical workflow (Dan's v1 scenario)

Time budget per step is realistic, not aspirational.

| Step | Action | Time |
|---|---|---|
| 1 | Pick **timber column** tool (T chord or palette tile) | 1 s |
| 2 | Click twice to place rectangle (600×340) — grain defaults to long axis | 3 s |
| 3 | Pick **steel plate** tool (P chord) | 1 s |
| 4 | Click twice to place plate over column face | 4 s |
| 5 | Open inspector, set plate thickness (e.g. 10 mm) | 3 s |
| 6 | Pick **start connection** action (chord ⇧C or palette) | 1 s |
| 7 | Click rear member (column) — confirms front=plate (auto-detected by overlap) | 2 s |
| 8 | Pick **HBS screw** tool (H chord). Defaults to last used spec, e.g. HBSPL12200 | 1 s |
| 9 | Click 6 times to place screws (or use grid placement — see §6 below) | 10 s |
| 10 | Pick **set load** action. Click twice for direction + enter magnitude | 5 s |
| 11 | Inspector shows live PASS/FAIL. Drag screws if FAIL. | 10–30 s |
| 12 | Toggle off `connections-meta` layer, export PDF | 5 s |
| **Total** | | **~45–60 s for a passing connection** |

If FAIL, the user iterates on step 11. Each drag updates the inspector and any red leaders in < 50 ms.

## 4. Grain direction UI

Grain is a property of the timber member. The user must always be able to see and edit it.

**Default**: when a TimberMember is placed by drawing a rectangle, grain = long axis of the rectangle.

**Visual cue**: hatch pattern on the timber outline showing parallel lines along the grain. Reuse `26-as1100-hatch.js` pattern primitive but at the timber-grain angle.

**Edit**:
- Inspector panel for selected TimberMember shows a "Grain direction" widget with 4 cardinal preset buttons (↑ ↓ ← →) and a numerical angle input (degrees from horizontal).
- A draggable grain‑direction arrow inside the timber outline lets the user rotate visually.
- When the grain rotates, the engine immediately recomputes all dependent checks for connections referencing this member.

---

## 5. Load arrow UI

The load is a property of the Connection. The user must always be able to see it.

**Visual**: a thick arrow originating at the centroid of the connection, length proportional to magnitude, with the magnitude label next to it (e.g. "25 kN ↓").

**Edit**:
- Inspector panel for selected Connection has a "Load" section: magnitude (kN), direction (4 cardinal presets + numerical angle), load case (dropdown: permanent / long / medium / short / instantaneous), service class (dropdown SC1/SC2/SC3).
- The arrow on canvas has a grip handle at its tail — drag to rotate; the inspector magnitude updates from the drag length (snapped to nice round kN values like 5, 10, 25).

**Important**: only ONE load per connection in v1. Multiple load cases is v1.x.

## 6. Screw placement modes

The user has three placement modes (toggle in toolbar when the screw tool is active):

### 6.1 Free-place (default)
Click to place a single screw. Snap to: edge of member, edge of plate, axis of existing screw (both grain-parallel and grain-perpendicular columns).

### 6.2 Grid place (v1)
After picking the tool, modifier key (Shift?) enters grid mode. Click twice for rectangle of N×M screws. Spacing defaults to 1.5× the minimum a₁ and a₂ for the active rule set (a "comfortable" passing layout).

Inspector shows N, M, a₁, a₂ as editable parameters; drag the grid corner to adjust.

### 6.3 Auto-place (v1.1)
"Given this many screws of this spec, find a layout that passes all checks." Solver minimises rectangle envelope. Out of v1 scope.

## 7. Live feedback during placement / drag

The single most important UX moment: while the user is dragging a screw, what do they see?

**On the canvas:**
- The dragged screw itself in cyan (active selection)
- The four edge distances from the dragged screw to the four edges of the rear member, **dimensioned in real-time**, with:
  - Green text + arrow if pass
  - Red text + arrow + bold if fail
- Pair-wise spacing lines to adjacent screws (only the closest 2–3 for clarity), showing Δu / Δv:
  - Green if pair passes
  - Red if pair fails
- A small "exclusion rectangle" around the dragged screw showing the (a₁, a₂) clearance envelope in the grain frame — semi‑transparent red fill, so the user sees where they *cannot* place this screw.

**In the inspector (right panel):**
- The header badge updates live: PASS / FAIL / WARN
- The check rows for the dragged screw highlight
- The capacity utilisation η updates live (rounds to 2 decimal places)

**In the status bar:**
- Live string: "Drag screw s4 — η=0.32 PASS" or "Drag screw s4 — a₃,t = 120 < 144 FAIL"

This is what makes the tool feel powerful. Manual drafting + manual checks loses every time to live feedback.

## 8. Static feedback (after drag ends, between edits)

When nothing is being dragged:

**On the canvas:**
- Each screw shows its **governing** edge distance only (the worst-case one — i.e. the smallest applied/required ratio per screw). Reduces clutter on detail-set output.
- All failing distances are shown with red leader + bold dimension.
- The connection's overall status badge near the load arrow: ✓ green or ✗ red.

**In the inspector:**
- Full table of all checks (per screw, four edges) — collapsible by default, expand on click.
- All pair-wise checks (passing ones collapsed).
- Capacity breakdown with multiplier trace.

**Toggle**: "Show all dimensions" toggle in the inspector flips between governing-only and all-edges view.

The inspector is `59-inspector.js`. For a Connection:

│   Rear:  TimberMember t1  (GL28h, 600×340 mm)           │

│                                                         │
│ ▸ Load                                                  │
│   F_d = 25.0 kN  ↓ (0, -1)                              │
│   α = 0° (parallel to grain)                            │
│   Load case: medium-term ▾                              │
│   Service class: SC1 ▾                                  │
│                                                         │
│ ▸ Screws (6)                                            │
│   All HBSPL12200                                        │
│   Common spec ▸ change                                  │
│                                                         │
│ ▸ Spacing & edge distance checks      [▾ details]       │
│   24 edge checks: 24 pass                               │
│   15 pair checks: 15 pass                               │
│                                                         │
│ ▸ Group reduction (n_ef)                                │
│   Row r1: 3 screws, a₁ = 60 mm = 5d → n_ef = 1.86       │
│   Row r2: 3 screws, a₁ = 60 mm = 5d → n_ef = 1.86       │
│                                                         │
│ ▸ Capacity                                              │
│   R_V,k base (p. 216, S=10 mm) = 12.99 kN/screw         │
│   × k_dens (GL28h, ρₖ=425) = 1.05  →  13.64             │
│   × k_mod (SC1, medium)    = 0.80  →  10.91             │
│   ÷ γM                      = 1.30  →  R_V,d = 8.39 kN  │
│   Group: 2 rows × n_ef 1.86 × 8.39  =  31.21 kN total   │
│   F_d / R_V,d = 25.0 / 31.21 = 0.80  →  η = 0.80  ✓     │
│                                                         │
│ ▸ Standards & version                                   │
│   Rothoblaas ETA-11/0030 (2019)                         │

│                                                         │

Every number traceable back to a clause or table. The engineer signing the detail must be able to audit the calc from the inspector alone.

## 10. Failure messaging — when checks fail

Failure messages should tell the engineer *exactly* what to do.

| Failure | Message |
|---|---|
| a₃,t fail | "Screw s4: distance to stressed end (bottom) = 80 mm. Required: 144 mm (12·d). Move screw 64 mm away from bottom edge." |
| a₁ pair fail | "Pair (s2, s3): grain‑parallel spacing = 35 mm < 42 mm (5·d·0.7). Increase spacing to ≥ 42 mm or stagger across grain to ≥ 25 mm (3·d·0.7)." |
| Capacity fail | "η = 1.45 — connection is overloaded by 45 %. Options: add 2 more screws, upgrade to HBSPL12200, reduce load to ≤ 22 kN." |
| Screw outside outline | "Screw s5 placed outside timber outline. Move inside the member or adjust member outline." |

Generated dynamically based on the magnitude of the shortfall.

## 11. Output for the detail sheet

When the user exports to PDF / DXF (existing `44-pdf-export.js` / `45-dxf-export.js`):

- The `connections-meta` layer is **excluded by default** from export (it's the live-check overlay, not the drawing).
- The dimension family that SHOULD export: a₁, a₂ between screws (one per row direction), edge distances (the governing one per direction), and the screw schedule (a small table off to the side: "6 × HBSPL12×200 @ 60 c/c × 100 c/c, predrilled Ø7").
- The screw schedule generation is a new helper (in `77-connection.js` or `79-checks-render.js`).

**Detail-sheet output goal**: looks at least as clean as STP 6011 details (per project root `CLAUDE.md`).

## 12. Failure modes the user will hit

- "Where's the grain direction set?" → ensure visible by default with hatch + arrow
- "Why is the load arrow pointing the wrong way?" → grip handle on tail; arrow goes from tail to head (head where the load applies)
- "I want to copy this connection to another sheet" → existing copy/paste handles entities; verify Connection serialises completely with its screws + load
- "The screws snapped to the plate but I wanted them snapped to a grid" → toggle between snap modes in the status bar
- "I changed the timber from GL24h to GL28h and the check changed slightly" → that's k_dens; show in inspector breakdown so the user understands why

## 13. Accessibility / readability

- Red is the primary fail colour. Engineers commonly have red-green colour-vision deficiency (~8% of males). Use **red + thicker line + ⚠ glyph** for fails, not red alone.
- Pass = green check, fail = red cross, warn = amber triangle. Glyphs first, colour second.
- Inspector text uses the existing app font; numerical values right-aligned in monospace where they're being compared (applied vs required).

---

## 14. Future UX considerations (v1.x / v2)

- **Multi-row drag**: shift-drag a whole row of screws while preserving a₁.
- **Constraint mode**: "lock a₁ = 65 mm and find the layout"; the engine treats a₁ as constrained and optimises within.
- **Capacity slider**: drag a slider to see "if F_d were X kN, would this still pass?"
- **Detail library**: save a passing connection as a reusable template (the V25 paper-space favourites strip exists for this — `56-favourites.js`).
- **Tabular detail compare**: side-by-side view of two designs (e.g. "HBSPL10×140 with 8 screws" vs "HBSPL12×200 with 6 screws") with cost/weight/capacity stats.
- **Specification output**: text block for the engineering notes: "Connection per ETA-11/0030. Screws: 6 × HBSPL12×200. Pre-drill Ø7. Min installation torque 50 Nm."
